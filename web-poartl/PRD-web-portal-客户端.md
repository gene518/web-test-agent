# Web Portal 客户端 PRD

基于仓库现状整理日期：2026-05-01

## 1. 文档目的

本文档用于定义 `web-test-agent` 的独立前端 Portal 客户端需求。该客户端将以 `web-poartl/` 作为前端工程目录，面向自动化测试场景提供项目选择、聊天驱动执行、节点过程可视化和历史会话查看能力。

本文档聚焦产品能力、交互规则、接口契约和验收口径，为前端实现和服务端 Portal API 实现提供统一需求基线。

## 2. 背景与目标

### 2.1 当前背景

- 当前仓库核心能力集中在 `web-agent/`，以 LangGraph 工作流驱动 `master`、`plan`、`generator`、`healer` 多阶段执行。
- 当前已有较完整的节点日志、模型调用日志、工具调用日志和自动化工程目录管理能力，但缺少面向业务用户的独立 Web 客户端。
- 当前调试方式更偏向 LangGraph Studio 和本地日志观察，不适合作为正式产品交互入口。

### 2.2 建设目标

- 为 `web-test-agent` 增加独立前端门户，前后端彻底分离。
- 使用接近 Codex 风格的深色三栏布局，降低用户理解和操作成本。
- 让用户可以直接在页面中完成自动化项目选择、聊天发起任务、查看阶段执行过程、切换历史会话。
- 将当前日志式过程观测升级为前端可消费的结构化会话轨迹。

### 2.3 核心目标

1. 用户首次进入时，不强制加载任何自动化项目。
2. 用户可主动打开自动化目录并选择一个自动化项目作为当前上下文。
3. 用户在未提前加载项目时，若通过聊天发起生成类任务，系统可在服务端识别或创建目标项目后自动刷新左侧文件树。
4. 用户可以在对话过程中查看每个 node 的执行过程、模型输出、决策信息、工具调用和工具结果。
5. 用户可以在页面最右侧查看跨会话历史记录，并切换查看旧会话内容。

## 3. 产品范围

### 3.1 本期范围

- 新建独立前端工程目录 `web-poartl/`。
- 提供三栏式 Web Portal 页面：
  - 左侧：当前自动化项目文件列表。
  - 中间：当前会话对话区与节点过程区。
  - 右侧：历史会话列表。
- 提供前后端分离的 Portal API 契约。
- 支持当前会话实时事件推送。
- 支持跨会话历史持久化浏览。

### 3.2 非目标

- 不做系统级目录选择器。
- 不做代码文件编辑器。
- 不做设置页。
- 不做模型切换。
- 不做附件上传。
- 不做历史会话搜索、标签筛选、删除。
- 不承诺服务重启后恢复尚未完成的 live interrupt 状态。

## 4. 用户场景

### 4.1 首次进入

用户首次打开 Portal 页面时，左侧文件区为空，不默认加载任何目录。页面应明确提示用户打开自动化目录，或直接通过聊天开始描述任务。

### 4.2 打开自动化目录

用户点击“打开自动化目录”后，前端调用服务端接口，读取 `DEFAULT_AUTOMATION_PROJECT_ROOT` 下的项目列表，并弹出项目选择视图。用户选择某个项目后，左侧加载该项目的文件树。

### 4.3 未加载项目直接发起生成

用户没有手动选择项目，但在聊天区直接要求生成测试计划或生成脚本。服务端应根据当前请求中的 `project_name`、`project_dir` 或执行后产物信息自动识别目标项目；若目标项目不存在但按现有流程已被创建，则在本轮执行结束后自动把该项目设为当前项目，并刷新左侧文件树。

### 4.4 查看节点过程

用户发送消息后，中间区域除了展示用户消息和助手总结消息，还应展示当前轮执行过程中的节点轨迹，包括节点开始、结束、路由决策、模型返回、工具调用、工具结果和异常信息。

### 4.5 查看历史会话

用户可在最右侧历史会话栏查看过去的聊天记录列表。点击某条历史记录后，中间区域切换展示该会话的完整内容与过程轨迹。历史会话默认只读，不允许继续发送新消息。

## 5. 页面信息架构

### 5.1 整体布局

Portal 页面采用三栏布局：

- 左栏：当前自动化项目区。
- 中栏：当前会话主工作区。
- 右栏：历史会话区。

整体视觉采用深色主题，保留高对比排版、紫色强调和终端感信息呈现，但不照搬参考图中无关功能。

### 5.2 左栏：文件区

左栏包含：

- 产品标题与简要副标题。
- 当前项目状态区。
- 文件树列表区。
- 底部“打开自动化目录”或“切换项目”按钮。

首次进入时：

- 不展示任何默认项目。
- 文件树区域显示空状态提示。

项目加载后：

- 仅展示当前自动化项目目录下的文件与文件夹。
- 不展示整个仓库。
- 不展示自动化根目录下其他项目内容。

### 5.3 中栏：主对话区

中栏包含：

- 当前会话标题区。
- 消息流区域。
- 每轮消息对应的节点过程区域。
- 输入框区域。

消息流应按时间顺序展示：

- 用户消息。
- 补参问题或中断提示。
- 助手最终总结消息。

若当前查看的是历史会话，中栏应切换到只读模式，输入框禁用并提示用户返回当前活跃会话继续交流。

### 5.4 右栏：历史会话区

右栏固定显示跨会话历史列表，包含：

- 会话标题。
- 关联项目名。
- 最近更新时间。
- 最近一条助手总结摘要。
- 当前状态，例如运行中、已完成、失败、只读。

历史会话按最近更新时间倒序展示，当前选中的会话需要高亮。

## 6. 功能需求

### 6.1 左侧文件树

#### 功能目标

展示“当前自动化项目”的文件结构，并在项目切换或项目自动识别后及时刷新。

#### 具体要求

- 首次进入不自动加载任何项目。
- 用户主动点击后再读取自动化根目录下的项目列表。
- 用户选中项目后，左侧展示该项目文件树。
- 文件树节点支持展开和收起。
- 点击文件时仅标记当前上下文文件，不要求内嵌编辑或预览。
- 当服务端在一次生成流程中自动创建或识别出目标项目时，左栏需要自动切换并刷新。

### 6.2 中间聊天区

#### 功能目标

承载用户任务输入、系统回复、补参交互和本轮执行结果展示。

#### 具体要求

- 用户可在底部输入框发起聊天。
- 若服务端当前会话处于 pending interrupt，前端继续发送文本时应视为补参内容。
- 对于当前活跃会话，消息发送后应实时显示执行进度。
- 历史会话默认只读，不支持继续发消息。
- 当前上下文文件路径可在输入框附近展示，但不影响服务端参数抽取逻辑。

### 6.3 节点过程展示

#### 功能目标

将当前工作流执行过程以结构化卡片形式展示给用户，替代纯日志式观察方式。

#### 具体要求

- 每一轮对话下展示对应的节点过程记录。
- 每个 node 至少展示以下信息：
  - `node_name`
  - 执行状态
  - 开始时间与结束时间
  - 路由或决策原因
  - 模型调用摘要
  - 工具调用摘要
  - 工具结果或错误摘要
- 节点过程默认可以折叠；展开后查看详细事件。
- 不展示模型内部隐藏推理内容。
- 过程展示应覆盖 `master`、`plan`、`generator`、`healer` 相关节点。

### 6.4 右侧历史会话

#### 功能目标

让用户能快速定位并查看过去的聊天记录。

#### 具体要求

- 历史会话采用跨会话持久化存储。
- 页面初始化时返回历史会话列表。
- 用户点击历史项后，中栏立即切换展示目标会话。
- 当前活跃会话需要在右栏中高亮。
- 历史会话默认只读。

## 7. 数据与接口契约

### 7.1 前后端关系

- 前端代码全部存放于 `web-poartl/`。
- 后端继续保留在 `web-agent/`。
- Portal 前端通过 REST + SSE 与服务端通信。
- 前端不直接依赖 LangGraph Studio 接口。

### 7.2 核心接口

#### 创建会话

- `POST /api/portal/sessions`
- 作用：创建新的活跃会话，并返回会话快照与历史列表。

#### 获取历史会话列表

- `GET /api/portal/history`
- 作用：返回历史会话摘要列表。

#### 获取单个会话详情

- `GET /api/portal/sessions/{sessionId}`
- 作用：返回指定会话的完整快照，包括消息、节点轨迹、当前项目和文件树。

#### 订阅会话实时事件

- `GET /api/portal/sessions/{sessionId}/stream`
- 作用：通过 SSE 推送当前会话的实时增量事件。

#### 发送消息

- `POST /api/portal/sessions/{sessionId}/messages`
- 作用：向活跃会话发送消息；若当前会话存在 pending interrupt，则内部按恢复补参处理。

#### 获取自动化项目列表

- `GET /api/portal/projects`
- 作用：读取 `DEFAULT_AUTOMATION_PROJECT_ROOT` 下可选项目。

#### 切换当前项目

- `POST /api/portal/sessions/{sessionId}/active-project`
- 作用：设置当前会话绑定的自动化项目，并返回最新文件树。

### 7.3 核心数据结构

#### PortalSessionSummary

- `sessionId`
- `title`
- `projectName`
- `updatedAt`
- `status`
- `lastAssistantSummary`

#### PortalSessionSnapshot

- `sessionId`
- `threadId`
- `activeProject`
- `fileTree`
- `messages`
- `turns`
- `pendingInterrupt`
- `runStatus`

#### PortalTurn

- `turnId`
- `userMessage`
- `assistantMessage`
- `stageSummaries`
- `nodeTraces`
- `createdAt`

#### NodeTrace

- `nodeName`
- `status`
- `routingReason`
- `startedAt`
- `finishedAt`
- `modelEvents`
- `toolEvents`

#### PortalEvent

- `type`
- `sessionId`
- `turnId`
- `sequence`
- `timestamp`
- `payload`

### 7.4 SSE 事件类型

本期至少支持以下 SSE 事件：

- `session_created`
- `message_started`
- `node_updated`
- `tool_updated`
- `project_changed`
- `message_completed`
- `message_failed`

## 8. 状态流与关键交互

### 8.1 新建会话

1. 前端初始化页面。
2. 调用创建会话接口。
3. 服务端返回一个新的活跃会话快照和历史会话列表。
4. 中栏展示空对话状态，右栏展示历史会话列表。

### 8.2 打开自动化目录

1. 用户点击“打开自动化目录”。
2. 前端获取自动化项目列表。
3. 用户选择某个项目。
4. 服务端返回当前项目文件树。
5. 左栏刷新为该项目内容。

### 8.3 发送消息

1. 用户在底部输入框发送文本。
2. 前端调用消息发送接口。
3. 同时通过 SSE 接收实时事件。
4. 中栏实时更新本轮消息、节点状态、工具调用和结果。
5. 会话结束后写入历史存储，并更新右栏排序。

### 8.4 中断补参

1. 服务端识别到缺少关键参数。
2. 当前会话进入 pending interrupt 状态。
3. 中栏展示补参问题。
4. 用户继续输入补充内容。
5. 前端仍调用消息发送接口，但服务端内部按 `resume` 流程处理。

### 8.5 自动识别或创建项目

1. 用户未提前选择项目，但发起计划生成或脚本生成任务。
2. 服务端执行过程中解析或创建目标项目目录。
3. 当目标项目可确定后，服务端推送 `project_changed` 事件。
4. 前端接收事件后刷新左栏文件树并更新当前项目显示。

### 8.6 切换历史会话

1. 用户点击右栏某条历史会话。
2. 前端请求该会话详情。
3. 中栏替换为该会话的消息和节点过程。
4. 输入框切换到只读提示状态。

## 9. 异常与边界

### 9.1 自动化根目录为空

- 若 `DEFAULT_AUTOMATION_PROJECT_ROOT` 下没有任何项目，项目选择视图应展示空状态提示。
- 不应导致页面报错或主流程阻断。

### 9.2 项目不存在

- 若历史会话关联的项目目录已被删除，会话详情仍应可查看。
- 左栏文件树区域应提示“项目目录不存在”。

### 9.3 SSE 断开

- 若实时连接中断，前端应提示连接异常，并允许用户手动重连或刷新会话详情。
- 已经完成的消息内容不得丢失。

### 9.4 工具执行失败

- 若工具调用失败，中栏节点过程应清晰展示错误摘要。
- 失败信息应进入当前轮记录，方便历史查看。

### 9.5 历史会话继续输入

- 历史会话默认只读。
- 用户在查看旧会话时不能继续发消息。
- 如需继续交流，应新建会话或切回当前活跃会话。

## 10. 验收标准

### 10.1 页面级验收

- 首次进入时左栏为空状态，不自动加载项目。
- 左栏可通过项目选择流程加载当前自动化项目文件树。
- 中栏可展示对话消息和节点过程。
- 右栏可展示跨会话历史列表。
- 点击历史会话后，中栏可切换到对应内容。

### 10.2 接口级验收

- 前端能通过 REST 接口创建会话、获取历史、获取项目、发送消息。
- 前端能通过 SSE 实时收到节点和工具事件。
- 项目自动识别后，前端能接收并处理 `project_changed` 事件。

### 10.3 端到端验收

- 用户可直接通过聊天生成测试计划，并在生成完成后看到自动加载的项目文件树。
- 用户可查看生成过程中的节点轨迹和工具调用结果。
- 用户可查看过去会话，并在页面内切换浏览。
- 当服务端触发缺参中断时，前端可继续补参并完成同一轮执行。

## 11. 实施说明

- 前端工程目录固定命名为 `web-poartl/`，保持与用户要求一致。
- 本文档是前端 Portal 的需求基线；后续如进入实现阶段，应进一步补齐前端工程脚手架、接口类型定义、状态管理方案和样式规范。
- 若后续服务端 Portal API 与本文档存在差异，应优先同步更新本文档，再进行联调。
